# Project Idea: Parcel Collection Tracking System

Sistem ini direka untuk memudahkan pengurusan dan penjejakan kutipan parcel secara digital, kemas, dan pantas.

## Peranan Pengguna

### 1. Admin (Pentadbir)
- **Akses**: Memerlukan akaun (username & password).
- **Fungsi**:
    - Log masuk ke sistem.
    - Memasukkan data parcel baharu (Nama Penerima, No. Telefon, No. Tracking).
    - Mengemaskini status kutipan parcel (contoh: "Pending" kepada "Collected").
    - Menguruskan rekod parcel yang ada.

### 2. User (Pengguna Biasa)
- **Akses**: Terbuka (tiada akaun diperlukan).
- **Fungsi**:
    - Membuat semakan status parcel sendiri.
    - **Data Input untuk Semakan**:
        1. Nama Penerima
        2. Nombor Telefon
        3. 4 digit terakhir nombor tracking
    - Melihat sama ada parcel mereka sudah sedia untuk dikutip atau telah diambil.

## Aliran Sistem
1. **Parcel Tiba**: Admin memasukkan maklumat ke dalam pengkalan data melalui dashboard.
2. **Semakan**: User melayari website, memasukkan butiran pengenalan diri, dan melihat status.
3. **Kutipan**: Apabila parcel diambil, Admin mengemaskini status dalam sistem.

## Objektif Projek
- Menggantikan sistem rekod manual kepada digital.
- Mempercepatkan proses semakan status bagi pengguna.
- Memastikan rekod kutipan parcel lebih tersusun dan mudah dikesan.
